Cap Defend V17d

문서 갱신: 2026-03-25 적용 버전: V17d 시간 기준: KST (미국장은 현지 정규장)

1. 핵심 규칙

  1. 자산 배분: 주식 60 : 코인 40 (실투입 58.8 : 39.2 : 현금 2%)
  2. 시그널 t-1 종가, 체결 t일 — Look-Ahead Bias 금지
  3. 매도 먼저, 매수 나중
  4. Delta 매매: 목표와 현재의 차이만큼만 거래
  5. 트리거(Crash/플립)는 즉시, 종목 교체는 앵커일에만
  6. rebalancing_needed 플래그: 목표 달성까지 재시도, 달성 후 방치
  7. 현금 2% 마진: 주식/코인 모두
현재 운영 상태: 코인 자동매매 운영 중 / 주식 자동매매 운영 중 (소액) / 텔레그램 알림 활성

5. 자동매매 시스템

5.1 모듈 역할

recommend_personal.py (09:15, 1일 1회) — 신호 생성

하는 일 안 하는 일
• 데이터 다운로드 (Yahoo, CoinGecko, Upbit)
• 전략 신호 계산 (카나리, Crash, Z-score, 헬스)
• signal_state.json 저장:
  추천 종목, Crash 상태, 플립 여부, VT 전일종가, 환율
• HTML 리포트 생성 (자산관리 페이지) 		매매 실행, 텔레그램 알림, 트랜치 관리, 상태 파일(trade_state) 수정

읽기 전용 모듈. 몇 번을 실행해도 매매 상태에 영향 없음.

--trade / --monitor — 매매 실행 (동일한 일을 수행)

공통 수행 작업
• signal_state.json 읽기 (freshness 체크 필요)
• 트리거 체크 → 조건 충족 시 즉시 매도:
  코인: 카나리 OFF, DD Exit(-25%), Blacklist(-15%)
  주식: VT Crash(-3%), 카나리 플립
• rebalancing_needed=true면 delta 매매 (미체결 취소 → 재계산 → 새 주문)
• 상태 파일 저장 (trade_state.json / kis_trade_state.json)
• 텔레그램 알림 (매매 실행 시)
--trade 전용 (1일 1회) --monitor 전용 (수시)
• 앵커일 체크 → 해당 트랜치 목표 갱신 + rebalancing_needed=true • 앵커 체크 안 함 (trade가 이미 처리)

trade와 monitor의 차이는 앵커 체크 유무뿐. 나머지는 동일.

주의: recommend가 실패하면 signal_state.json이 갱신 안 됨 → trade/monitor가 이전 데이터로 동작. recommend 실패 시 로그 확인 필요.
향후 개선: trade+monitor를 하나의 executor로 통합, 앵커 체크를 별도 분리 검토 중.

키 의미 예시

trade_state.json (코인) / kis_trade_state.json (주식) — 매매 상태

키 의미 예시

tranches 각 트랜치의 종목/비중/앵커월 {"1": {"picks": ["GLD","EEM","PDBC"], "anchor_month": "2026-03", ...}}

5.5 리밸런싱 플래그

주식(kis_trade_state.json)과 코인(trade_state.json)은 별도 상태 파일에서 독립 관리.

true가 되는 상황: 포트폴리오 목표가 바뀔 때

  • 앵커일 도달 → 트랜치 종목 갱신 (목표: 새 종목 33%씩)
  • 카나리 플립 → 공격/방어 전환 (목표: 새 포트폴리오)
  • Crash → 전량 매도 (목표: 보유 0주, 전액 현금)
  • DD Exit / Blacklist → 해당 코인 매도 (목표: 해당 코인 0)

false가 되는 상황: 목표 달성 (금액 ±5% 이내 or 주수 일치)

Crash든 리밸런싱이든 동일. "현재 보유가 목표와 맞으면 false."

false이면: 다음 앵커까지 가격 변동 무시. 매매 안 함.

8. 용어집

용어 설명
카나리 시장 추세를 판단하는 선행 지표. 위에 있으면 공격(Risk-On), 아래면 방어(Risk-Off)
Hysteresis (히스테리시스) 노이즈 필터. 전환 시 ±N% 밴드를 넘어야 상태가 바뀜
EW (동일비중) 선정된 종목을 같은 비중으로 보유
Z-score 모멘텀과 Sharpe를 표준화하여 합산한 점수. 높을수록 우선
Crash Breaker (급락 차단) 기준 지수가 급락하면 전량 매도하고 일정 기간 현금 대기
DD Exit (낙폭 이탈) 보유 종목이 최근 고점 대비 크게 하락하면 매도
Blacklist (급락 제외) 하루에 크게 빠진 종목을 일정 기간 매수 대상에서 제외
PFD (플립 후 재평가) 카나리 전환 후 N일 뒤 종목을 다시 확인하여 리밸런싱
트랜치 자산을 N등분하여 서로 다른 날에 리밸런싱. 타이밍 리스크 분산
앵커일 각 트랜치가 리밸런싱하는 날짜 (예: 1일, 8일, 15일, 22일)
rebalancing_needed 목표 포트폴리오와 현재가 다를 때 true. 달성하면 false
Delta 매매 전량 매도→재매수가 아닌, 목표와 현재의 차이만큼만 거래
TX 백테스트에서 가정한 편도 수수료+슬리피지 (주식 0.2%, 코인 0.4%)
