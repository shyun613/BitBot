2.2 카나리 신호

EEM(신흥국 ETF)이 200일 이동평균 위에 있으면 Risk-On(공격), 아래면 Risk-Off(방어).

0.5% Hysteresis: 전환 시 ±0.5% 밴드로 노이즈 필터링.

2.3 종목 선정

Z-score = zscore(12M 모멘텀) + zscore(Sharpe 252일)

Z-score 상위 3개 종목을 33%씩 균등 배분.

Sharpe가 포함되어 변동성이 큰 종목(PDBC 등)은 자연스럽게 패널티를 받음.

2.4 방어 모드

Risk-Off 시 IEF, BIL, BNDX, GLD, PDBC 중 6개월 수익률 Top 3을 EW로 보유.

2.5 Crash Breaker

VT(글로벌 주식) 일간 수익률 ≤ -3% → 전량 매도 (공격+방어 모두) → 3영업일 현금 대기 후 재진입.

Crash/플립은 예외: 전 트랜치를 동시에 즉시 처리합니다.

5.3 Monitor 역할

Monitor는 단순 미체결 확인이 아닙니다.
rebalancing_needed=true면 능동적으로 매매합니다: 미체결 전량 취소 → 목표 재계산 → 새 주문 → 완료 체크
rebalancing_needed=true rebalancing_needed=false
--trade 앵커 갱신 + 매매 앵커 체크만 (매매 없음)
--monitor 미체결 취소 + 재계산 + 매매 미체결 확인만

코인 monitor 추가 역할: Crash(BTC -10%), 카나리 OFF, DD Exit(-25%) 긴급 매도. DD Exit은 해당 코인만 개별 매도 (전량청산 아님).

5.5 안전장치 및 상태 관리

장치 설명
원자적 저장 signal_state.json: tmp + os.replace (부분쓰기 방지)
flock run_trade.sh 래퍼로 --trade와 --monitor 동시실행 방지
완료 판단 금액 ±5% 이내 OR 주수 일치 (소액에서 1주 차이 허용)
매도 실패 복구 Crash/Canary-off 후 잔고 확인, 잔량 있으면 rebalancing_needed=true 유지
수량 추적 before ± delta 기준 (절대 보유수량 아님, 과매도 방지)

5.6 리밸런싱 플래그 사이클

앵커일 도달 → 해당 트랜치 목표 갱신 → rebalancing_needed: true
  → --trade 매매 시도 (주문 → 5초 → 취소 → 재시도 x5)
  → 목표 ≈ 현재 (±5% or 주수 일치) → rebalancing_needed: false
  → 미완료 → --monitor가 장중 매시 재시도
  → false면 다음 앵커까지 가격 변동 무시

Crash → 전량 매도 → 잔량 있으면 true 유지 → 쿨다운 3일 (매매 차단)
플립 → 전 트랜치 즉시 전환 → rebalancing_needed: true
쿨다운 중 → 앵커일이 와도 리밸런싱 차단

6. 운영 가이드

6.1 평소 (할 일 없음)

텔레그램 알림이 올 때만 대응. 코인은 서버가 자동, 주식은 자동매매 안정화 후 자동.

6.2 텔레그램 알림 시 (연 3~5회)

페이지 열기 → 매일 확인 섹션 → 지시대로 매도/매수 예약 (자동매매 미안정 시)
자동매매 안정화 후: 서버가 자동 처리, 텔레그램은 확인용

6.3 자산 관리 (월 1회 또는 수시)

자산 관리 섹션 열기
→ 주식-신한 수동 입력 + 한투 조회 + 코인 조회 + 환율 조회
→ 계산 (60:40 비중 + ETF 주수)
→ 스냅샷 저장

9. 핵심 규칙 요약

  1. 시그널: t-1일 종가, 체결: t일 (Look-Ahead Bias 금지)
  2. 매도 먼저, 매수 나중
  3. Delta 매매: 차이만큼만 거래 (전량 매도→재매수 아님)
  4. 트리거와 리밸런싱 분리: Crash/플립은 즉시, 종목 교체는 앵커일에만
  5. rebalancing_needed 플래그: 목표 달성까지 재시도, 달성 후 방치
  6. 현금 2% 마진: 주식/코인 모두
  7. 한 번에 한 가지만 변경: 전략 연구 시 baseline 고정
