5.2 주문 실행 패턴 (주식+코인 공통)

핵심: 주문 → 대기 → 미체결 취소 → 재시도 (최대 5회) 
for attempt in 1..5:
  1. 현재가 조회
  2. 주문 (주식: 지정가 매수+1%/매도-1%, 코인: 시장가 분할)
  3. 5초 대기
  4. 미체결 전량 취소
  5. 잔고 조회 → 체결량 확인
  6. 목표 달성? → 종료
  7. 미달 → 다음 시도 (새 현재가로)
주식 (KIS) 코인 (업비트)
주문 방식 지정가 (매수 +1%, 매도 -1%) 시장가 분할매매
재시도 최대 5회, 5초 간격 분할 루프 내부
수량 추적 before ± delta (과매도/과매수 방지) 금액 기반 잔여량
장 시간 23:30~06:00 (정규장만) 24시간

5.3 --trade vs --monitor 상세

주식 (auto_trade_kis.py)

--trade (23:35, 1일 1회) --monitor (00:05~06:05, 매시)
앵커 체크 오늘이 앵커일(1/8/15/22)이고 이번 달 미처리면 → 트랜치 목표 갱신 + rebalancing_needed=true 앵커 체크 안 함
트리거 체크 Crash, 카나리 플립, Crash 복귀 → 즉시 매도/전환 트리거 체크 안 함 (09:15 신호 기준)
매매 실행 rebalancing_needed=true면 delta 매매 rebalancing_needed=true면 미체결 취소 → 재계산 → 새 주문
rebal=false일 때 앵커 날짜만 확인, 해당 안 되면 아무것도 안 함 미체결 확인만, 없으면 즉시 종료

코인 (auto_trade_upbit.py)

--trade (09:20, 1일 1회) --monitor (:05/:35, 30분마다)
앵커 체크 오늘이 앵커일(1/11/21)이고 이번 달 미처리면 → 트랜치 목표 갱신 앵커 체크 안 함
트리거 체크 Crash, 카나리 플립, DD Exit, 헬스 실패, PFD5, buffer 변경 긴급 트리거: Crash(BTC -10%), 카나리 OFF(BTC < SMA60×0.99), DD Exit(-25%)
매매 실행 트리거 있으면 분할매매 (시장가) 긴급 시 즉시 매도 + pending 복구 + rebalancing_needed 재시도
DD Exit 전 트랜치에서 해당 코인 제거 + 매도 해당 코인만 개별 매도 (전량청산 아님)

rebalancing_needed가 true로 바뀌는 상황

상황 주식 코인
앵커일 도달 해당 트랜치 목표 갱신 → true 해당 트랜치 목표 갱신 → true (트리거로 매매)
카나리 플립 전 트랜치 즉시 전환 → true 전 트랜치 즉시 전환 → true
Crash 복귀 전 트랜치 재진입 → true (3일 윈도우 자동 해소)
Crash/Canary-off 매도 실패 잔량 있으면 true 유지 pending에 저장 → true
DD Exit / 헬스 실패 - 트랜치 제거 + true

rebalancing_needed가 false로 바뀌는 상황

상황 설명
목표 달성 금액 ±5% 이내 OR 주수 일치
Crash 전량 매도 성공 잔고 0 확인 → false
Canary-off 전량 매도 성공 잔고 0 확인 → false
pending 모두 완료 (코인) monitor에서 false로 해제

5.5 안전장치 및 상태 관리

장치 설명

5.6 리밸런싱 플래그 사이클 (주식+코인 공통)

주식과 코인은 별도 상태 파일에서 독립적으로 플래그를 관리합니다.
주식: kis_trade_state.json / 코인: trade_state.json 
[정상 리밸런싱]
앵커일 도달 → 해당 트랜치 목표 갱신 → rebalancing_needed: true
  → --trade 매매 시도 (주문 → 5초 → 취소 → 재시도 x5)
  → 목표 ≈ 현재 (±5% or 주수 일치) → false
  → 미완료 → --monitor가 재시도 (주식: 장중 매시, 코인: pending 복구)
  → false면 다음 앵커까지 가격 변동 무시

[Crash]
전량 매도 → 잔량 있으면 true 유지 (monitor가 재시도)
  → 잔량 없으면 false → 쿨다운 기간 동안 매매 차단
  → 주식: 3영업일 고정 / 코인: 최근 3일 윈도우 자동 해소

[카나리 플립]
전 트랜치 즉시 전환 → true → 매매 후 false

[코인 전용]
DD Exit → 해당 코인만 매도 + 트랜치 제거 → true
  → 빈 비중은 현금 유지 → 다음 앵커일에 새 종목으로 채움
PFD5 → 플립 후 5일에 전 트랜치 재갱신 → true

6. 운영 가이드

6.1 평소 (할 일 없음)

텔레그램 알림이 올 때만 대응. 코인은 서버가 자동, 주식은 자동매매 안정화 후 자동.

6.2 텔레그램 알림 시 (연 3~5회)

페이지 열기 → 매일 확인 섹션 → 지시대로 매도/매수 예약 (자동매매 미안정 시)
자동매매 안정화 후: 서버가 자동 처리, 텔레그램은 확인용

6.3 자산 관리 (월 1회 또는 수시)

자산 관리 섹션 열기
→ 주식-신한 수동 입력 + 한투 조회 + 코인 조회 + 환율 조회
→ 계산 (60:40 비중 + ETF 주수)
→ 스냅샷 저장

9. 핵심 규칙 요약

  1. 시그널: t-1일 종가, 체결: t일 (Look-Ahead Bias 금지)
  2. 매도 먼저, 매수 나중
  3. Delta 매매: 차이만큼만 거래 (전량 매도→재매수 아님)
  4. 트리거와 리밸런싱 분리: Crash/플립은 즉시, 종목 교체는 앵커일에만
  5. rebalancing_needed 플래그: 목표 달성까지 재시도, 달성 후 방치
  6. 현금 2% 마진: 주식/코인 모두
  7. 한 번에 한 가지만 변경: 전략 연구 시 baseline 고정
