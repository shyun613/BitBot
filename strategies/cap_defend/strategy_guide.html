Cap Defend V17

문서 갱신: 2026-03-26 적용 버전: V17 시간 기준: KST (미국장은 현지 정규장)

1. 핵심 규칙

  1. 자산 배분: 주식 60 : 코인 40 (각 executor 내부에서 마진 관리)
  2. 시그널 t-1 종가, 체결 t일 — Look-Ahead Bias 금지
  3. 매도 먼저, 매수 나중
  4. Delta 매매: 목표와 현재의 차이만큼만 거래
  5. 트리거(Crash/플립)는 즉시, 종목 교체는 앵커일에만
  6. rebalancing_needed 플래그: 목표 달성까지 재시도, 달성 후 방치
  7. 마진: 주식 2% (고정), 코인 coin_trade_state에서 설정 (전체 현금 배분 없음)
현재 운영 상태: 코인 자동매매 운영 중 / 주식 자동매매 운영 중 (소액) / 텔레그램 알림 활성

2. 전략 상세 (주식 vs 코인)

주식은 월 4회 분산 매매 + VT 급락(-3%) 방어. 코인은 월 3회 분산 + 개별 코인 낙폭(-25%) 손절 + 급락(-15%) 제외.

4.1 시나리오 A: 평범한 일상

서버가 코인(매 10분) + 주식(매 30분, 장중)을 자동 매매. 아무것도 안 해도 됩니다.

4.2 시나리오 B: 긴급 알림 수신

시스템이 자동으로 손절 또는 전환을 마쳤다는 뜻. 증권사/업비트 앱에서 체결만 확인. 개입 불필요.

에러 알림: 로그 확인 후 수동 개입 여부 판단.

4.3 시나리오 C: 밴드 이탈 시 자산 비율 맞추기

포트폴리오 페이지 → 주식(한투 조회) + 코인(업비트 조회) → 계산.
주식 비중이 55~65% 밖이면 계좌 간 자금 이동.

5. 자동매매 시스템

recommend가 오늘의 추천 + HTML 리포트를 만들고, executor가 트랜치 관리 + 가드 + 매매를 수행.

이 장은 시스템 장애 대응이나 코드 수정 시 참고하는 기술 명세입니다.

5.1 아키텍처

설계 원칙: recommend = 순수 신호 (트랜치 무관, trade_state 안 읽음) / executor = 트랜치 관리 + 가드 + 매매 (단일 모드)
recommend_personal.py [1일 1회, 09:15]
시장 데이터 → 공격/방어 추천 종목 계산 → signal_state.json 저장 → HTML 리포트
↓ signal_state.json (recommend만 씀)
executor (코인) [cron 수시]
가드 → 앵커/플립 → merge → Delta 매매
executor (주식) [cron 수시]
가드 → 앵커/플립 → merge → Delta 매매
↓ coin_trade_state.json / kis_trade_state.json (executor만 씀)
api_server.py [백그라운드, 조회 전용]

5.2 모듈별 역할

recommend [순수 신호 생성기, 1일 1회]

1. 시장 데이터 수집

  • Yahoo Finance, CoinGecko, Upbit에서 가격 데이터 다운로드
  • 로컬 캐시에 저장

2. 전략 신호 계산

  • 카나리 판단: EEM > SMA200 (주식), BTC > SMA60 (코인), 히스테리시스 포함
  • 공격 종목 선정: Z-score Top 3 (주식), 시총순 Top 5 (코인)
  • 방어 종목 선정: 6M return Top 3 (주식), 현금 (코인)
  • 헬스 필터: Mom30>0, Mom90>0, Vol90≤5% (코인)

3. 가드 기준값 계산

  • 각 코인의 peak_60d (60일 종가 최고점) — DD Exit용
  • 각 코인의 prev_close (전일 종가) — Blacklist용
  • VT prev_close, sma10 — Crash 발동/복귀용

4. 출력

  • signal_state.json 저장 (추천 종목 + 기준값)
  • HTML 리포트 생성 (운영자 확인용)
안 하는 일: 계좌 주문, trade_state 쓰기, Crash 발동/복귀 판단, 트랜치/앵커 관리
(예외: HTML 표시용 cash_buffer 조회만 coin_trade_state에서 읽음)
몇 번 실행해도 매매 상태에 영향 없음
executor [단일 모드, cron 수시]

1. 데이터 읽기

  • signal_state.json 읽기 (추천 종목, 카나리, 기준값)
  • coin_trade_state.json / kis_trade_state.json 읽기 (트랜치, 가드 상태)
  • 거래소 API로 현재가 + 잔고 조회 (주식은 환율 포함)

2. 가드 체크 (현재가 vs signal 기준값 비교)

  • Crash: 현재가 ≤ signal.vt_prev_close × 0.97 → 전량 매도 → crash_active=true → 3일+SMA10 대기 후 복귀
  • DD Exit: 현재가 ≤ signal.peak_60d × 0.75 → 해당 종목만 매도 → 현금 유지 → 다음 앵커에서 새 종목
  • Blacklist: 현재가 ≤ signal.prev_close × 0.85 → 해당 종목 매도 + 7일 제외 → 다음 앵커에서 새 종목

3. 트랜치 관리

  • 카나리 플립: signal.risk_on ≠ trade_state.prev_risk_on → 전 트랜치를 공격/방어로 즉시 전환
  • PFD: 플립 후 5일 경과 + pfd_done=false → 전 트랜치 picks를 signal 최신값으로 재평가 (코인만)
  • 앵커 체크: 오늘 ≥ 앵커일 AND anchor_month ≠ 이번달 → 해당 트랜치에 signal.picks 반영 (하루 1회만)

4. 매매 실행

  • 전 트랜치 merge → 최종 target 포트폴리오
  • target vs 현재 잔고 → Delta > 임계치면 매매 (매도 먼저, 매수 나중)
  • 목표 달성 시 rebalancing_needed=false

5. 저장 + 알림

  • coin_trade_state.json / kis_trade_state.json 갱신
  • 텔레그램 알림 (매매/가드 발동/에러)
안 하는 일: 전략 신호 계산 (카나리/Z-score/헬스), signal_state 수정
api_server [백그라운드]
잔고/수익률 조회 API 제공. 운영자는 recommend가 만든 HTML 리포트만 보면 됨. 매매/신호 개입 없음.

5.3 executor 실행 흐름

run_once():
  signal = read_signal_state()
  state  = read_trade_state()

  1. 미체결 전량 취소 → 잔고 재조회
  2. 가드 체크 (최우선, 기준값은 signal에서 종가 기준으로 계산, 발동은 장중 현재가)
     - Crash(주식): 현재가 ≤ vt_prev_close × 0.97 → 전량 매도, 3일+SMA10 대기 후 복귀
     - DD Exit(코인): 현재가 ≤ peak_60d × 0.75 → 해당 종목 매도 → 현금 → 다음 앵커에서 새 종목
     - Blacklist(코인): 현재가 ≤ prev_close × 0.85 → 해당 종목 매도 + 7일 제외 → 다음 앵커
  3. 카나리 플립: signal.risk_on ≠ state.prev_risk_on → 전 트랜치 즉시 전환
  4. PFD: flip_date + 5달력일 이후 첫 실행에서 1회 수행 (코인만)
  5. 앵커 체크: risk_on이면 offense, 아니면 defense picks를 해당 트랜치에 반영
     (anchor_month 체크로 하루 1회만 적용)
  6. 전 트랜치 merge → 최종 target
     산식: 각 트랜치 비중을 1/N (코인 1/3, 주식 1/4)로 합산
     * crash_active=true → target=100% 현금 (트랜치 무시)
     * exclusions에 있는 종목 → target에서 제거, 해당 비중은 현금
     * recommend가 BL 코인을 여전히 picks에 넣더라도 executor가 걸러냄
  7. target vs 잔고 → Delta > 임계치면 매매
  8. trade_state 저장 + 텔레그램 알림

이벤트 우선순위: Crash > DD/BL > 플립 > PFD > 앵커 > Delta 매매

실행 스케줄 (KST)

모듈 스케줄 비고
recommend 09:15 1일 1회, signal_state 저장
코인 executor 09:20 이후 매 10분 + 랜덤 0~90초 지연 24시간 상시 (코인 시장 무휴)
주식 executor 23:35 이후 매 30분 + 랜덤 0~90초 지연 미국 정규장 시간만

코인 executor와 주식 executor는 각각 독립 flock으로 자기 자신의 동시실행만 방지.
코인과 주식은 서로 다른 거래소/상태파일이므로 동시에 돌아도 충돌 없음.

주문 실행 패턴

매도 우선, Delta 기준, 최대 5회 재시도
for attempt in 1..5:
  1. 현재가 조회
  2. 지정가 주문 (±0.3%)
  3. 5초 대기 → 미체결 전량 취소
  4. 잔고 재조회 (before ± delta 추적)
  5. 목표 달성 → 종료 / 미달 → 다음 시도

매수 금액 부족 시:
  - 매도 먼저 실행하여 현금 확보 후 매수 진행
  - 그래도 부족하면 비중 비례로 축소 매수 (가능한 만큼만)
  - 미달분은 rebalancing_needed=true 유지, 다음 실행에서 재시도

5.4 상태 관리

signal_state.json = 오늘의 추천 (recommend만 씀)  |  coin_trade_state.json = 코인 집행 상태  |  kis_trade_state.json = 주식 집행 상태
각 executor가 자기 상태 파일만 읽고 씀. recommend는 trade_state를 건드리지 않음.

5.5 안전장치 & 장애 대응

장치 설명
원자적 저장 signal/trade_state 모두 tmp + os.replace
flock executor 동시실행 방지
stale signal updated_at이 24시간 이상이면 매매 스킵, 가드만 체크
멱등성 같은 날 여러 번 실행해도 앵커/PFD 중복 안 됨 (anchor_month 체크)
수수료 버퍼 주식 2% 버퍼, 코인은 coin_trade_state의 cash_buffer 반영
최소 주문 Delta가 최소 주문 금액 미만이면 스킵

recommend 자동 복구

장치 설명
자동 재시도 cron에서 recommend 실패 시 5분 후 1회 자동 재실행
stale signal 경고 executor가 updated_at 24시간 초과 감지 → 텔레그램 "signal 갱신 안 됨" 알림
이전 signal 유지 recommend 실패해도 executor는 이전 signal_state로 가드 체크 계속 (매매는 보류)
watchdog 5분마다 recommend 프로세스 + signal_state 타임스탬프 확인, 이상 시 텔레그램 알림
executor 실패: rebalancing_needed=true 유지 → 다음 cron에서 자동 재시도 (코인 10분, 주식 30분).
가드 응답 시간: executor 주기적 실행으로 급락 감지 (코인 최대 10분, 주식 최대 30분). 이는 cron 기반 한계이며 실시간 웹소켓 감시로 개선 가능 (로드맵).

7. 용어집

용어 설명
카나리 시장 추세를 판단하는 선행 지표. 위에 있으면 공격(Risk-On), 아래면 방어(Risk-Off)
Hysteresis (히스테리시스) 노이즈 필터. 전환 시 ±N% 밴드를 넘어야 상태가 바뀜
EW (동일비중) 선정된 종목을 같은 비중으로 보유
Z-score 모멘텀과 Sharpe를 표준화하여 합산한 점수. 높을수록 우선
Crash Breaker (급락 차단) 기준 지수가 급락하면 전량 매도하고 일정 기간 현금 대기
DD Exit (낙폭 이탈) 보유 종목이 최근 고점 대비 크게 하락하면 매도
Blacklist (급락 제외) 하루에 크게 빠진 종목을 일정 기간 매수 대상에서 제외
PFD (플립 후 재평가) 카나리 전환 후 N일 뒤 종목을 다시 확인하여 리밸런싱
트랜치 자산을 N등분하여 서로 다른 날에 리밸런싱. 타이밍 리스크 분산
앵커일 각 트랜치가 리밸런싱하는 날짜 (예: 1일, 8일, 15일, 22일)
rebalancing_needed 목표 포트폴리오와 현재가 다를 때 true. 달성하면 false
Delta 매매 전량 매도→재매수가 아닌, 목표와 현재의 차이만큼만 거래
TX 백테스트에서 가정한 편도 수수료+슬리피지 (주식 0.2%, 코인 0.4%)
